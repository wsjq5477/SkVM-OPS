# Text Skill Compiler Quick Start

This tool optimizes text-analysis skills, especially log parsing skills. It uses an LLM as a compiler to generate a deterministic Python analyzer, tests it locally, repairs failures with the LLM when needed, and emits an optimized Claude skill.

## Inputs

You need:

- A source skill or workflow file, for example `path/to/SKILL.md` or `path/to/workflow.js`.
- One or more sample text/log files.
- An output directory for the optimized skill.
- An LLM provider. For local models, use an OpenAI-compatible endpoint.

## Local Model Setup

Your local model server must expose the OpenAI Chat Completions API:

```text
POST /v1/chat/completions
```

The response must contain:

```json
{
  "choices": [
    {
      "message": {
        "content": "{\"files\":{...}}"
      }
    }
  ]
}
```

Recommended local serving options include vLLM, LM Studio, llama.cpp server, Ollama OpenAI-compatible endpoint, or any gateway that supports `/v1/chat/completions`.

For best results, use a model with:

- Strong JSON following.
- At least 32k context; 64k is better for logs and long skills.
- Good code generation, because the model must generate Python scripts and tests.

## Compile With a Local Model

Example:

```powershell
python doc-skill-ops\compiler\text_skill_compiler.py `
  --skill path\to\your\SKILL.md `
  --sample path\to\sample_log.txt `
  --out doc-skill-ops\outputs\your-skill-optimized `
  --name your-skill-optimized `
  --provider local `
  --base-url http://127.0.0.1:8000/v1 `
  --model your-local-model-name
```

If your local server requires a key:

```powershell
python doc-skill-ops\compiler\text_skill_compiler.py `
  --skill path\to\your\SKILL.md `
  --sample path\to\sample_log.txt `
  --out doc-skill-ops\outputs\your-skill-optimized `
  --name your-skill-optimized `
  --provider local `
  --base-url http://127.0.0.1:8000/v1 `
  --model your-local-model-name `
  --api-key your-local-key
```

If no key is provided, the compiler sends `Authorization: Bearer local`.

## Output

The compiler writes:

```text
<out>/
  SKILL.md
  scripts/
    analyze_text.py
  references/
    analysis-contract.md
    report-format.md
  tests/
    test_analyze_text.py
  compiled/
    flow_graph.raw.json
    flow_graph.json
    graph_validation.json
    contract.json
    solidification.json
    generation.json
    validation.json
  evaluation/
    baseline-vs-optimized.json
    baseline-vs-optimized.md
```

Use `<out>/SKILL.md` as the optimized Claude skill.

## How It Optimizes

The LLM compiler:

1. Reads the original skill/workflow and sample log.
2. Extracts a SkillFlow execution graph from the natural-language workflow.
3. Validates the graph and derives the input/output contract from it.
4. Identifies deterministic graph nodes and operations, such as parsing, regex extraction, grouping, timeline generation, and schema validation.
5. Generates `scripts/analyze_text.py` and tests.
6. Runs generated tests locally.
7. If tests fail, sends failures back to the LLM for repair.
8. Writes an optimized `SKILL.md` that runs the Python analyzer first and leaves residual reasoning to Claude.

Python is not the intelligence layer. Python is the deterministic execution layer generated by the LLM.

## Validation

Run all local tests:

```powershell
python -m unittest discover -s doc-skill-ops\tests
```

Run the generated skill tests inside an output directory:

```powershell
python -m unittest discover -s doc-skill-ops\outputs\your-skill-optimized\tests
```

Run the generated analyzer:

```powershell
python doc-skill-ops\outputs\your-skill-optimized\scripts\analyze_text.py `
  doc-skill-ops\outputs\your-skill-optimized\samples\sample_log.txt `
  --handoff --pretty
```

## Troubleshooting

- If JSON parsing fails, use a stronger model or lower temperature on your local server. The compiler expects raw JSON, not markdown.
- If generated tests fail, increase `--max-repair-rounds`.
- If the local model times out, reduce sample size or use a longer-context/faster serving backend.
- If token reduction is small, make sure the generated script emits compact handoff JSON instead of full raw logs.
- If your local endpoint path is not `/v1`, pass the exact base URL that makes `/chat/completions` valid after appending it.
