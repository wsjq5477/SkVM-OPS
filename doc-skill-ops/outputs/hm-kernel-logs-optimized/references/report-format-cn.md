# 报告格式

最终答案必须使用固定报告格式。

# 1、故障摘要

记录故障类型、崩溃线程、CPU、关键寄存器和一句关键结论。

# 2、关键日志

列出与崩溃直接相关的核心日志，包含时间和原始日志内容。

# 3、时间线

按时间升序列出主要事件。

# 4、根因分析

区分事实、推理和不确定点。脚本检测到的 root-cause candidate 只能作为候选，Claude 需要根据日志证据重新判断。

# 5、建议排查方向

按优先级给出工程排查动作、目标和预期结果。

## 禁止事项

- 不要输出 JSON 作为最终报告。
- 不要输出 `<think>`。
- 不要添加第六个一级标题。
- 不要忽略 analyzer 已提取的确定性事实。
